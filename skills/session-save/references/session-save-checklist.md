# Session Save — Checklist

Этот чеклист определяет обязательные поля и критерии качества для файлов памяти сессии.

## Обязательные поля файла сессии

Каждый файл памяти сессии (`session_YYYY-MM-DD.md`) должен содержать:

| # | Поле | Описание | Пример |
|---|------|----------|--------|
| 1 | **Дата** | ISO 8601 | `2026-04-09` |
| 2 | **Проект** | Путь к репозиторию или название | `/home/user/projects/my-app` |
| 3 | **Ветка** | Текущая git-ветка | `feature/auth-flow` |
| 4 | **Резюме** | Что сделано за сессию (2-5 предложений) | «Реализовал JWT-аутентификацию, написал тесты для login/register эндпоинтов» |
| 5 | **Ключевые решения** | Решения с обоснованием (почему, а не только что) | «Выбрали Redis для хранения сессий, потому что уже используется для кеша» |
| 6 | **Изменённые файлы** | Список файлов (или `git diff --stat`) | `src/auth/router.py`, `tests/test_auth.py` |
| 7 | **Блокеры / открытые вопросы** | Что мешает продолжить | «Нужен API-ключ для Stripe от заказчика» |
| 8 | **Следующие шаги** | Конкретные действия (actionable) | «1. Добавить refresh token. 2. Настроить CORS. 3. Написать миграцию для таблицы sessions» |
| 9 | **Неочевидный контекст** | Всё, что потеряется без записи | «Заказчик попросил НЕ использовать OAuth — только email/password. Устно, не в ТЗ» |

## Формат файла памяти

```markdown
---
name: session_2026-04-09
description: "Краткое описание для индекса MEMORY.md"
type: project
---

## Дата
2026-04-09

## Проект
/home/user/projects/my-app (ветка: feature/auth-flow)

## Что сделано
- Реализовал JWT-аутентификацию (login, register, refresh)
- Написал 12 тестов для auth модуля
- Настроил Redis для хранения сессий

## Ключевые решения
- **Redis для сессий** — уже используется для кеша, не нужен отдельный сервис
- **python-jose вместо PyJWT** — поддерживает JWS/JWE, совместим со стеком проекта

## Изменённые файлы
- src/auth/router.py (новый)
- src/auth/schemas.py (новый)
- src/auth/dependencies.py (новый)
- tests/test_auth.py (новый)
- src/core/config.py (добавлены JWT_SECRET, JWT_ALGORITHM)

## Блокеры
- Нужен API-ключ Stripe от заказчика для интеграции платежей

## Следующие шаги
1. Добавить refresh token endpoint
2. Настроить CORS для фронтенда
3. Написать миграцию для таблицы sessions
4. Интегрировать auth middleware в существующие роуты

## Неочевидный контекст
- Заказчик устно попросил НЕ использовать OAuth — только email/password
- В Directus уже есть таблица users, но мы создаём отдельную для приложения
```

## Именование файлов

- Формат: `session_YYYY-MM-DD.md`
- При нескольких сессиях в день: `session_YYYY-MM-DD_2.md`
- Расположение: `~/.claude/projects/<project-hash>/memory/`

## Обновление MEMORY.md

После создания файла сессии — обязательно добавить строку в `MEMORY.md`:

```markdown
- [Сессия 2026-04-09](session_2026-04-09.md) — JWT-аутентификация, Redis для сессий, блокер: ключ Stripe
```

Формат: `- [Заголовок](файл.md) — краткое описание (до 150 символов)`

## Критерии качества

- **Резюме**: конкретно, не «работал над проектом», а «реализовал X, починил Y»
- **Решения**: всегда с «почему», не просто «выбрали Redis»
- **Следующие шаги**: actionable — можно начать выполнять сразу, не надо додумывать
- **Неочевидный контекст**: всё, что нельзя узнать из кода или git log

## Active-session lockfile (v1.5.0)

Помимо файла `session_YYYY-MM-DD.md` и обновления `MEMORY.md`, `/session-save`
обязан писать **`.active-session.lock`** в тот же memory dir. Цель — дать
`pre-flight-check.sh` хуку в **других** параллельных Claude-сессиях сигнал,
что этот проект уже в работе, и предотвратить дублирование задач
(инцидент NeuroExpert 2026-04-11: один и тот же tech-debt deploy.sh
был исправлен дважды параллельно).

Формат (JSON, всегда rewrite — не append):

```json
{
  "timestamp": 1712845200.0,
  "pid": 12345,
  "branch": "feat/my-feature",
  "project": "/home/user/projects/example",
  "note": "Саммари сессии до 150 символов"
}
```

Правила:
- **timestamp** — Unix epoch seconds. `pre-flight-check.sh` игнорирует locks
  старше 10 минут (stale).
- **pid** — pid текущего процесса (`$$` / `os.getpid()`).
- **branch** — из `git branch --show-current`; `"unknown"` если не git-репо.
- **project** — абсолютный путь к рабочей директории.
- **note** — та же строка, что идёт в MEMORY.md как краткое описание.

Lockfile — идемпотентный маркер «я был здесь недавно», не mutex. Не блокирует
параллельные сессии, только предупреждает их. Stale locks игнорируются
автоматически — никаких cleanup задач не нужно.
