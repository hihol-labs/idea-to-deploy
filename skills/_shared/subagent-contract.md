# Субагентский контракт (v1.85.0)

Канонический блок для ЛЮБОГО Agent-запуска из скиллов методологии.

**Почему это отдельный слой:** защиты основного цикла (хуки, env-нормы
settings.json) НЕ транспортируются в субагентов автоматически. Live-замер
2026-07-11 (упражнение «диагностическая петля», итерация 1): субагент сжёг
2 попытки на WindowsApps-шиме python, хотя в основном цикле этот класс отказа
был закрыт ещё в v1.84.0 (ENV_PROBE_RE). Единственный надёжный транспорт —
текст промпта субагента. Скилл, порождающий агента, вставляет секции этого
файла в промпт как есть.

## 1. Env-преамбула (вставлять в промпт каждого субагента на Windows-хосте)

> Замечания по среде (правила проекта):
> - Python-скрипты запускать ТОЛЬКО через обёртку
>   `sh "$HOME/.claude/skills/_shared/itd_py.sh" <script.py> [args]` — она
>   обходит WindowsApps-заглушку и сама форсирует `PYTHONIOENCODING=utf-8`
>   + `PYTHONUTF8=1`. Голые `python`/`python3` не использовать.
> - Консоль Windows — cp1251: печать не-ASCII символов вне запускателя может
>   падать с UnicodeEncodeError; при прямых вызовах интерпретатора добавляй
>   `PYTHONIOENCODING=utf-8` в env.

На Linux/WSL-хосте преамбула безвредна (запускатель прозрачен) — вставляй её
безусловно, ветвление по ОС в промпте не нужно.

## 2. Приёмка сдачи субагента (evidence-gate)

Заявление субагента «готово» без evidence — не готово. В промпт субагента,
выполняющего изменение кода/конфига, вставляй требование:

> Прежде чем заявлять «готово»: прогони верификацию изменённого поведения
> (typecheck/линт по правилам репозитория + минимальный self-test новых веток
> поведения) и приложи вывод команд. Что не удалось прогнать — явно пометь
> «НЕ ПРОВЕРЕНО» с причиной. Временные тест-файлы удали перед завершением.

Принимающая сторона (скилл/основной цикл) при получении результата: если в
ответе нет ни evidence, ни пометки «НЕ ПРОВЕРЕНО» — сдача не принимается,
результат перепроверяется своими командами или возвращается субагенту.
Live-обоснование (упражнение «verification gap», 2026-07-11): 3 из 5
микрозадач сданы вообще без верификации (процессный gap 60%); протокол с
этим требованием дал self-test 16 кейсов + eslint против 8 кейсов без него.
Калибровка: для read-only/исследовательских сдач evidence = цитаты/пути
источников; полную лестницу верификации на микрозадачах не требуй
(фактический gap там 0% — данные того же упражнения).
