# MCP Docs — Checklist

Чеклист для подтягивания **актуальной** документации библиотек/фреймворков через
MCP-провайдеров документации (в первую очередь Context7), когда устаревшее или
угаданное знание API может повлиять на реализацию.

> Read-only по дизайну: скилл читает внешние доки и возвращает выжимку в stdout
> или в рабочие заметки/план/ревью-отчёт. Файлы проекта он не меняет.

## Основной источник

Предпочитай MCP-инструменты документации, особенно Context7 (если доступен):

1. **Резолвни library ID** для пакета/фреймворка/платформы.
2. **Задай узкий вопрос** по конкретной задаче реализации.
3. **Зафиксируй** source library ID и принятое решение в рабочих заметках, плане
   или ревью-отчёте.

Если MCP-документация недоступна — используй официальную документацию проекта как
fallback и явно укажи, что MCP-источник был недоступен.

## Использовать ПЕРЕД

- Добавлением или апгрейдом зависимостей.
- Написанием интеграционного кода против SDK, auth-провайдеров, платёжных API,
  деплой-платформ, UI-фреймворков, ORM, очередей, browser API.
- Починкой сбоев, которые могут быть вызваны version drift.
- Выбором фреймворк-специфичных конвенций в `/bugfix`, `/refactor`, `/infra`,
  `/deploy`, `/deps-audit`, `/harden`.

## Форма вывода

```text
DOC SOURCE:
VERSION OR LIBRARY ID:
DECISION:
IMPLEMENTATION IMPACT:
FOLLOW-UP RISK:
```

## Правила

- Не слать секреты, проприетарные сниппеты, креды или приватные данные клиентов в
  инструменты документации.
- Предпочитай официальные или высокорепутационные источники.
- Держи запросы узкими — ровно под текущую задачу.
- Не блокируй крошечные локальные фиксы, если поведение API уже доказано тестами
  или кодом.
- Если доки конфликтуют с конвенциями репозитория — объясни конфликт и предпочти
  конвенцию репозитория, если она не сломана и не deprecated.

## Self-validation (перед выводом)

- [ ] Library ID резолвнут (или указано, что MCP недоступен → fallback).
- [ ] Запрос узкий, под конкретную задачу.
- [ ] Source library ID и решение зафиксированы в заметках/плане/ревью.
- [ ] Секреты/проприетарный код не отправлены наружу.
- [ ] Конфликт «доки vs конвенция репо» (если есть) явно разобран.
