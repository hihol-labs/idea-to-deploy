# Руководство пользователя

## Быстрый старт

Скажите Claude Code что-нибудь вроде:
- "Хочу создать сервис доставки еды"
- "Новый проект — CRM для стоматологии"
- "Сделай Telegram-бот для записи на маникюр"

Claude Code спросит, какой вариант вам подходит, и направит по нужному маршруту.

## Три маршрута

### А) Полный цикл (идея → деплой)
Для тех, кто хочет получить работающий продукт.

Что произойдёт:
1. Claude задаст 3-5 вопросов о проекте
2. Создаст документацию (стратегия, архитектура, план)
3. Построит структуру проекта
4. Напишет код шаг за шагом
5. После каждой фичи — автоматические тесты
6. Задеплоит

Время: 4-12 часов (зависит от сложности)
Когда выбирать: вы точно знаете что хотите и готовы строить.

### Б) Только планирование
Для тех, кому нужно сначала согласовать план.

Что произойдёт:
1. Claude задаст вопросы
2. Создаст 6 документов
3. Остановится — код не пишет

Время: 20-40 минут
Когда выбирать:
- Нужно показать план заказчику
- Хотите передать проект другому разработчику
- Не уверены в деталях — хотите сначала посмотреть план

После: можете сказать "начинай реализацию" → переход к маршруту А.

### В) У меня есть документация
Для тех, у кого уже есть PROJECT_ARCHITECTURE.md или IMPLEMENTATION_PLAN.md.

Что произойдёт:
1. Claude прочитает ваши документы
2. Создаст CLAUDE_CODE_GUIDE.md — пошаговые промпты

Время: 5-10 минут
Когда выбирать: документация уже написана (вами или другим ИИ).

## Скиллы для ежедневной работы

Эти скиллы работают автоматически во время разработки:

| Ситуация | Что сказать | Какой скилл |
|----------|------------|-------------|
| Нашли баг | "не работает: [ошибка]" | /bugfix |
| Нужны тесты | "напиши тесты для auth.py" | /test |
| Код тормозит | "тормозит загрузка страницы" | /perf |
| Код запутанный | "упрости этот файл" | /refactor |
| Не понимаете код | "объясни как работает auth" | /explain |
| Нужна документация | "создай README" | /doc |

## Советы

1. **Начните с маршрута А** если не знаете что выбрать — он включает в себя Б
2. **Большие проекты** лучше разбивать на фазы: сначала MVP, потом расширение
3. **Если Claude забегает вперёд** — скажите "стоп, только шаг N"
4. **Если что-то пошло не так** — скажите "откати последний шаг"
5. **Коммитьте после каждого шага** — это позволяет откатиться


## Рекомендуемые модели

| Скилл | Минимум | Рекомендуется | Почему |
|-------|---------|---------------|--------|
| /project | Sonnet | Sonnet | Простая маршрутизация |
| /blueprint | Sonnet | Opus | Генерация архитектуры требует глубокого анализа |
| /kickstart | Sonnet | Opus | Полный цикл — максимальное качество критично |
| /guide | Sonnet | Opus | Качество промптов зависит от понимания контекста |
| /review | Sonnet | Opus | Поиск несоответствий требует сильной модели |
| /test | Sonnet | Sonnet | Достаточно для генерации тестов |
| /bugfix | Sonnet | Sonnet | Достаточно для анализа ошибок |
| /perf | Sonnet | Opus | Оптимизация требует глубокого понимания |
| /refactor | Sonnet | Sonnet | Достаточно для рефакторинга |
| /explain | Haiku | Sonnet | Объяснение не требует сложных рассуждений |
| /doc | Sonnet | Sonnet | Достаточно для документации |

**Совет:** Для первого прогона нового проекта используйте Opus на этапах blueprint/kickstart/review. Для ежедневной работы (test, debug, refactor) Sonnet достаточно.
